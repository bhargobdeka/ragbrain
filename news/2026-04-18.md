🧠 Learn & Understand

• Guided Determinism (Salesforce's term, but the pattern is universal): Fixed routing/handoff rules with bounded LLM reasoning in between. It's the practical answer to the "how much autonomy do I give agents?" question in production. Study this pattern — it's how real systems stay auditable without becoming fully hard-coded. Futurum writeup
• Non-human identity governance: Salesforce's "Trusted Agent Identity" and the governance gap research both point to the same blind spot — most IAM systems weren't built for agents acting on behalf of users. Understanding how agent identity, least-privilege, and audit trails intersect is increasingly a core skill for anyone building production agentic systems.
• MCP (Model Context Protocol) as an enterprise integration standard: Now administered by the Linux Foundation, MCP is being used as a discovery and interop layer across Bedrock, Foundry, Salesforce, and more. If you haven't dug into MCP's spec and how it handles context propagation across agent hops, now's the time.

───

🛠️ Build & Practice

• Build a minimal "guided determinism" router: Take a multi-agent flow you already have and carve out a deterministic routing layer — fixed rules decide which agent gets called, while LLMs only handle reasoning within each agent. Even a quick LangGraph or custom orchestrator sketch will make the pattern concrete and interview-ready.
• Experiment with MCP tool exposure: If you have any local APIs or tools, try wrapping them as MCP-compatible endpoints and connecting them to an agent runtime (Claude Desktop, Cursor, or a custom harness). The enterprise push means this will be a practical integration skill very soon.
